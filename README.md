\## üìò Project Description



The \*\*Customer Ledger dApp\*\* is a decentralized application built on the Flare Coston2 test network.  

It enables users and businesses to maintain a tamper-proof, fully on-chain ledger of customers.  

Each customer record includes:



\- A blockchain address  

\- A customer name  

\- A balance that can be increased or decreased  



The ledger ensures high transparency, immutability, and decentralization‚Äîideal for bookkeeping, credit systems, or tracking user balances securely.

\## ‚≠ê Features



\### \*\*1. Add Customer Records\*\*

\- Register a customer with:

&nbsp; - Customer wallet address  

&nbsp; - Customer name  

\- Stores information directly on the blockchain



\### \*\*2. Retrieve Customer Details\*\*

\- Fetch name and balance for any registered customer

\### \*\*3. Update Customer Balance\*\*

\### \*\*5. Transaction Feedback System\*\*

\- Handles errors gracefully  



\### \*\*6. Clean \& Minimal Frontend Integration\*\*

\## üí° How It Solves Real Problems



Traditional ledger systems rely heavily on centralized databases that:



\- Can be manipulated  

\- Can be lost due to corruption  

\- Require trust in the server owner  

\- Lack transparency for auditing  



This project solves these issues:



\### \*\*Immutable Customer Records\*\*

Once stored on the blockchain, customer data cannot be altered maliciously.



\### \*\*Decentralized Balance Updates\*\*

Balances are updated through verified transactions, making them traceable and tamper-proof.



\### \*\*Transparent Auditing\*\*

Any user can inspect the records via contract functions or blockchain explorers.
